A 3-tiered design-in-browser workflow for Twitch extensions
This is a POC project to demonstrate my 3-step design-in-browser workflow as well as a tiered CSS approach that aims to be reusable while allowing artistic freedom.
Tier 1: layout the responsive screens using BassCSS
  • After sketching out the wireframes and user flow on paper, I use BassCSS (MIT License) to build the fluid and responsive html pages.
  • Benefits:
    • The file size of BassCSS is only 14kb. Its utility CSS classes are very comprehensive. It's possible to build fluid layout without writing any additional CSS, thus the html pages become very reusable.
    • BassCss has documentation and code snippets.
    • Separating the layout CSS from the style CSS makes the layout less prone to breaking and much easier to maintain.
    • With the utility class names, the CSS becomes very readable.
    • It's much easier to create professional-looking UI with good alignment, padding, margin, etc. with something like BassCSS or other CSS frameworks than building everything from scratch.

Tier 2: reusable CSS for generic styling
  • The goal of this step of the workflow is to create a set of styling CSS that is reusable for future Twitch extension projects.
  • This is the set of CSS that will become the "style guide" and needs time to be built out and tested cross browsers, so together with BassCSS it can be shared with the community as boilerplates.
  • The examples below add prefix "e-" to the class names of this Tier 2 CSS to differentiate from the BassCSS class names

  • The examples above assume that creator can customize:
    • a dark brand color (used for background)
    • a tone slightly lighter than the brand color (used for the alternative background color)
    • a bright accent color
    • a Google font, letter spacing, font style (applied to .bold in the examples)
    • having .rounded or not
    • the opacity of color variables: --c-text, --c-text-alt and --c-text-alt-2
    • There should be some kind of auto detect feature: if the accent color is dark enough, the text of the Button and the Tabs will use .e-c-white; if not, .e-c-black
